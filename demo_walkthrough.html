Titan Maintenance Intelligence Platform — a 5-agent crew turning one sensor alert into a maintenance decision
Run with: streamlit run app.py
LLM: Gemini Flash-Lite

1
The big picture
One alert comes in. Five specialist agents look at it from different angles. One decision comes out.
Titan's plants fire 22,000+ sensor alerts a day with no prioritization. The demo shows how an agentic system fixes that: instead of a human reading raw telemetry, a team of AI agents perceives the alert, reasons about how risky and costly it really is, checks it against safety policy, and returns a single prioritized, explainable recommendation — deciding on its own whether it can act autonomously or must escalate to a human.
The trigger event (this is what kicks everything off)
{
  "event_type": "SENSOR_THRESHOLD_BREACH",
  "asset_id":   "CNC-Lathe-07",
  "plant":      "Plant 3 - Valencia",
  "signal":     "spindle_vibration",
  "value":      0.48,        // measured
  "threshold":  0.35,        // safe limit  →  37% over
  "timestamp":  "2026-06-20T07:14:33Z"
}
You set this in the Streamlit sidebar and press ▶ Run the crew. Everything below happens automatically.
2
The pipeline — step by step
The agents run as a sequential pipeline: the three specialists assess first, the governance gate reviews, and the orchestrator synthesizes last. Each step's output feeds the next.
⚡
0 · Event received
Streamlit calls run_pipeline(event) in titan_agents.py, which builds the 5 agents and the task list.
📡
1 · Reliability Agent — how sick is the machine?
Reads telemetry, scores the anomaly, checks repair history, estimates Remaining Useful Life (RUL) and a health score. → e.g. health 29/100, AT_RISK, RUL ~44h.
🔍
2 · Process Agent — can we even fix it in time?
Checks spare-parts stock, lead time, technician availability and recurring process delays. Surfaces the biggest bottleneck and a process_risk score.
💰
3 · Cost Agent — how much money is at stake?
Calculates downtime cost ($180,000/day), orders at risk (Airbus, Rolls-Royce…), ROI of fixing now vs later, and a financial_impact_score.
🛡️
4 · Safety & Governance Agent — is the action allowed?
Runs each proposed action through policy_validator. Draft work order / notify / update dashboard → APPROVED. Machine shutdown → REQUIRES_APPROVAL. LOTO override → REJECTED.
⚙️
5 · Orchestrator — the final call
Reads all four assessments, checks for conflicts, computes the priority score, applies the safety override if needed, decides AUTONOMOUS vs ESCALATE, and executes the approved actions (drafts the work order, notifies the lead, updates the dashboard).
✅
Output
A single JSON decision with priority score, risk level, recommended action, rationale, work-order draft and next check-in — shown as cards in the UI.
The priority formula the Orchestrator uses:
Priority = (failure_probability × 0.4) + (financial_impact_score × 0.35) + (process_risk × 0.25). If Safety & Governance flags something the formula misses, safety overrides the number.

Five agents, each with a narrow job and its own set of tools. No single agent reasons across everything — that's the whole point.

💰Cost Intelligence AgentSpecialist

Agents decide, tools execute
The agents (the LLM) do the thinking. The 12 tools just fetch data or perform an action — they never decide anything.
In a real deployment these tools would hit live SCADA sensors, the maintenance system (CMMS), the parts warehouse and the BMC Helix portal. For the demo they're mocked in step2_mock_tools.py — they return realistic, deterministic data so the agents reason over a believable scenario without needing real factory infrastructure.

5
What you'll see on screen
The Streamlit UI in app.py.
As it runs
A progress bar advances 1/5 → 5/5. Each specialist's card fills in live with its headline numbers (risk, health, RUL, $/day, gate decision) the moment that agent finishes.
The final card
The Orchestrator's decision: priority score, risk level, a big 🤖 AUTONOMOUS or 🔺 ESCALATE badge, any conflict it found, the recommended action, the rationale, and the work-order draft.
Why it takes ~3 minutes: the free Gemini tier allows ~15 requests/minute. A full run makes ~30 LLM calls, so we throttle to MAX_RPM=10 — CrewAI deliberately pauses between calls so we never hit the rate limit. Slower on purpose, but it never crashes.
